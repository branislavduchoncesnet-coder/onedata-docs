All features are described in official [onedata documentation](https://onedata.org/#/home/documentation/25/user-guide/interfaces/web-file-browser.html).  
Following below are only essentials and organization specifics cases.

* Log into [Onedata portal](https://onedata.e-infra.cz) via one of the identity provider.

## Overview

The Web File Browser in Onedata provides a simple way to **manage files in your spaces directly from a browser**.  
It works like a typical file manager with folders and files.

- Data is organized in **spaces (virtual directories)**  
- You can **upload, download, browse, and manage files**  
- Interface is accessible without installing additional tools  

---

## Basic Navigation

### Browse files
- Open a **space**
- Navigate using folders (like a standard file system)
- Click folders to enter them

### Open files
- **Double-click** a file to open or download it  

---

## File Operations

### Upload files
- Open target directory
- **Drag & drop files into the browser window**  

### Download files
- Double-click file  
- Or use download option from context menu

### Create folders
- Use **“New folder”** option in UI

### Delete / Rename
- Use file **context menu (right-click or actions menu)**

---

## File Management Features

### Organizing data
- Files are structured in:
  - Spaces
  - Directories
  - Files

### Metadata
- You can attach metadata to:
  - Spaces
  - Folders
  - Files  

---

## Sharing & Access

### Access control
- Permissions are managed per space or file
- Controlled via users and groups

### Public sharing
- Files or data can be shared with others
- Access depends on assigned permissions

---

## Tips & Notes

- Allow **pop-ups in browser** for proper file handling  
- For large-scale or advanced work, consider:
  - CLI tool (**Oneclient**)
  - APIs (REST, S3, CDMI)  

---

## When to Use Web File Browser

Use it when you want to:
- Quickly upload/download files
- Browse data without setup
- Perform basic file operations

Use other interfaces when you need:
- Automation
- Large data transfers
- Programmatic access
